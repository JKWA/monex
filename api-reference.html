      Monex.Either

      The Monex.Either module provides an implementation of the Either monad, which represents values that can either be Right (success) or Left (error).

      Monex.Either.Left

      Represents the Left variant of the Either monad, used to model an error or failure.

      Monex.Either.Right

      Represents the Right variant of the Either monad, used to model a success or valid result.

      Monex.Eq

      The Monex.Eq protocol defines an equality function, equals?/2, for comparing two values.

      Monex.Foldable

      The Monex.Foldable protocol defines two core folding operations: fold_l/3 (fold left) and fold_r/3 (fold right).

      Monex.Identity

      The Monex.Identity module represents the identity monad, where values are simply wrapped in a structure and operations are applied directly to those values.

      Monex.LazyTask

      The Monex.LazyTask module provides an implementation of a lazily evaluated task monad. This monad allows for deferred execution of tasks, wrapping a function that can be executed asynchronously and awaited later.

      Monex.LazyTaskEither

      The Monex.LazyTaskEither module provides an implementation of the LazyTaskEither monad, which represents asynchronous computations that can either be Right (success) or Left (failure).

      Monex.LazyTaskEither.Left
      Monex.LazyTaskEither.Right
      Monex.Maybe

      The Monex.Maybe module provides an implementation of the Maybe monad, representing optional values as either Just (a value) or Nothing (no value).

      Monex.Maybe.Just

      Represents the Just variant of the Maybe monad, used to model the presence of a value.

      Monex.Maybe.Nothing

      Represents the Nothing variant of the Maybe monad, used to model the absence of a value.

      Monex.Monad

      The Monex.Monad protocol defines the core monadic operations: ap/2, bind/2, and map/2.

      Monex.Operators
      Monex.Ord

      The Monex.Ord protocol defines a set of comparison functions: lt?/2, le?/2, gt?/2, and ge?/2.
